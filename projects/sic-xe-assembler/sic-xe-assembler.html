SIC-XE Assembler

A two-pass assembler implementation for the SIC/XE architecture with symbol table generation and object code production

Project Overview

This project implements a comprehensive two-pass assembler for the SIC/XE architecture, capable of processing assembly language source files and generating object code with symbol tables. It demonstrates advanced compiler theory concepts and assembly language processing techniques.

Technologies Used

C++ STL Assembly Language Compiler Theory Two-Pass Assembler Symbol Tables

Key Features

  • Two-Pass Assembly: First pass builds symbol table, second pass generates object code
  • SIC/XE Architecture: Full support for SIC/XE instruction formats and addressing modes
  • Symbol Table Management: Comprehensive symbol and literal table handling
  • Multiple Instruction Formats: Support for Type 1, Type 2, Type 3, and Type 4 instructions
  • Addressing Modes: Immediate, indirect, direct, and indexed addressing support
  • Directive Processing: Handles START, END, BYTE, WORD, RESW, RESB, BASE directives

How It Works

  1. First Pass: Reads source file, builds symbol table, calculates addresses
  2. Symbol Resolution: Processes labels, literals, and assembler directives
  3. Address Calculation: Determines memory locations for all symbols
  4. Second Pass: Generates object code using resolved symbols
  5. Output Generation: Creates listing file (.l) and symbol table file (.st)
  6. Error Handling: Validates syntax and reports assembly errors

Architecture Support

SIC/XE Instruction Formats

  • Type 1: 1-byte instructions (FIX, FLOAT, HIO, etc.)
  • Type 2: 2-byte instructions (ADDR, CLEAR, TIXR, etc.)
  • Type 3: 3-byte instructions with various addressing modes
  • Type 4: 4-byte instructions for extended addressing

Addressing Modes

  • Immediate: # symbol (operand is the value)
  • Indirect: @ symbol (operand is the address)
  • Direct: symbol (operand is the value at address)
  • Indexed: symbol,X (adds index register value)

Usage Example

# Compile the project
make

# Run with sample assembly file
./asxe P2sample2.sic

# View generated listing file
cat P2sample2.l

# View generated symbol table
cat P2sample2.st

PROG START 2000
LDA ALPHA

GAMMA RESW 1

Technical Implementation

Core Components

  • pass1.cpp: First pass implementation for symbol table generation
  • pass2.cpp: Second pass implementation for object code generation
  • optab.cpp: Opcode table and mnemonic table management
  • asxe.cpp: Main program entry point and file processing

Data Structures

  • SYMTAB: Symbol table mapping symbols to addresses
  • LITTAB: Literal table for literal pool management
  • OPTAB: Opcode table for instruction format information
  • MNEMONICTAB: Mnemonic to opcode mapping
