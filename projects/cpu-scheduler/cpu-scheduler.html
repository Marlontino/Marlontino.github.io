CPU Scheduler

A multi-threaded CPU scheduling simulator implementing Shortest Job Next (SJN) algorithm

Project Overview

This project implements a sophisticated CPU scheduler simulator that manages processes with alternating CPU and I/O bursts using the Shortest Job Next (SJN) algorithm. It features multi-threaded execution, exponential averaging for burst time estimation, and comprehensive process management.

Technologies Used

C++ Multi-threading POSIX Threads Scheduling Algorithms Process Management Memory Management

Key Features

  • Shortest Job Next (SJN): Processes scheduled based on shortest CPU burst time
  • Exponential Averaging: Optional burst time estimation using exponential averaging
  • I/O Burst Handling: Supports processes with alternating CPU and I/O bursts
  • Multi-threaded Implementation: Uses POSIX threads for concurrent execution
  • Busy Waiting: Implements thread synchronization with busy waiting
  • Memory Management: Proper dynamic memory allocation and deallocation

How It Works

  1. Process Loading: Reads process burst data from input files
  2. Burst Analysis: Parses alternating CPU and I/O burst times
  3. Scheduling Decision: SJN algorithm selects process with shortest CPU burst
  4. Multi-threaded Execution: Concurrent processing using POSIX threads
  5. Burst Estimation: Optional exponential averaging for future burst prediction
  6. Performance Tracking: Monitors turnaround time, wait time, and completion status

Scheduling Algorithm

Shortest Job Next (SJN)

  • Processes are sorted by their next CPU burst time
  • The process with the shortest CPU burst is executed first
  • When exponential averaging is enabled, estimated burst times are used for sorting

Exponential Averaging Formula

τ(n+1) = α × t(n) + (1-α) × τ(n)

Where:
• τ(n+1): Estimated burst time for next CPU burst
• t(n): Actual burst time of current CPU burst
• τ(n): Previous estimated burst time
• α: Exponential averaging factor (0.0 < α < 1.0)

Usage Example

# Compile the project
make clean && make

# Run with basic SJN scheduling
./schedule bursts.txt

# Run with exponential averaging (α = 0.5)
./schedule bursts.txt 0.5

Input File Format

# bursts.txt - One line per process, space-separated integers
4 7 6
3 8 3 1 6
5 2 3

# Represents:
# Process 0: CPU=4, I/O=7, CPU=6
# Process 1: CPU=3, I/O=8, CPU=3, I/O=1, CPU=6
# Process 2: CPU=5, I/O=2, CPU=3
