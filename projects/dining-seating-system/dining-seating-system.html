Dining Seating System

A multi-threaded producer-consumer system simulating restaurant seating management with robot concierges

Project Overview

This project implements a sophisticated multi-threaded producer-consumer system that simulates a restaurant's seating management using robots. It demonstrates advanced concurrent programming concepts with thread-safe queue management and business rule enforcement.

Technologies Used

C++ Multi-threading POSIX Threads Producer-Consumer Thread Safety Mutex & Condition Variables

Key Features

  • Producer Robots (Greeters): Generate table requests for General Tables and VIP Rooms
  • Consumer Robots (Concierges): T-X and Rev 9 robots process seating requests
  • Thread-Safe Queue: Mutex-protected queue with condition variables
  • Business Rules: Maximum 18 total requests, maximum 6 VIP requests
  • FIFO Processing: First In, First Out request handling
  • Configurable Timing: Adjustable sleep times for all threads

How It Works

  1. Producer Threads: Generate table requests with configurable timing
  2. Queue Management: Thread-safe queue enforces business constraints
  3. Consumer Threads: T-X and Rev 9 robots process requests from queue
  4. Business Rules: Monitor class enforces request limits and priorities
  5. Real-time Logging: Comprehensive logging of all operations
  6. Statistics Reporting: Final report shows production and consumption totals

System Architecture

Producer Threads

  • General Table Producer: Generates general table requests
  • VIP Room Producer: Generates VIP room requests
  • Rate Limiting: Configurable sleep times between requests
  • Graceful Termination: Stops after reaching request limit

Consumer Threads

  • T-X Robot: Processes requests from the queue
  • Rev 9 Robot: Processes requests from the queue
  • FIFO Processing: Maintains request order
  • Continuous Operation: Runs until all requests are processed

Usage Example

# Compile the project
make clean && make

# Basic usage with 50 requests
./dineseating -s 50

# Advanced usage with custom timing
./dineseating -s 100 -g 50 -v 200 -x 75 -r 150

# Quick test with minimal requests
./dineseating -s 5

Command Line Options

  • -s <number>: Total number of seating requests (default: 100)
  • -g <milliseconds>: General Table producer sleep time (default: 100ms)
  • -v <milliseconds>: VIP Room producer sleep time (default: 100ms)
  • -x <milliseconds>: T-X consumer sleep time (default: 100ms)
  • -r <milliseconds>: Rev 9 consumer sleep time (default: 100ms)

Technical Implementation

Monitor Class

Thread Synchronization

  • Mutex Protection: Prevents race conditions in shared data structures
  • Condition Variables: Efficient producer-consumer synchronization
  • Memory Management: Automatic cleanup in destructors

Business Rules

  • Queue Limits: Maximum 18 unconsumed requests at any time
  • VIP Constraints: Maximum 6 VIP room requests in queue
